## 📁 1. Estrutura do Projeto e Artefatos

O projeto está organizado da seguinte forma para facilitar a navegação:

```text
qa-manual-api-portfolio/
│
├── test-plan/              # Estratégia, escopo e planejamento macro dos testes
├── test-cases/             # Casos de teste detalhados (Caminhos felizes, tristes e limites)
├── bug-reports/            # Documentação e reports de bugs encontrados (padrão Jira)
├── exploratory-testing/    # Notas e cartas de testes exploratórios (Session-Based Testing)
├── postman/                # Collections e Environment do Postman para testes de API
├── sql/                    # Queries SQL utilizadas para validação de massa de dados
├── evidences/              # Prints e gravações de tela das execuções e bugs
├── checklists/             # Listas de verificação rápida (Sanity e Smoke Tests)
└── mindmaps/               # Mapas mentais para brainstorming de cenários de teste
```
